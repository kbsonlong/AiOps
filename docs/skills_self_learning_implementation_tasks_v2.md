# AIOps Skills 自学习系统实施任务分解 (v2.0)

## 概述

本文档基于修正后的《AIOps Skills 自学习系统设计 (v2.0)》方案，拆解为具体的可执行任务。主要调整：简化架构、聚焦MVP、优化任务依赖关系。

## 1. 总体实施路线图（修正版）

```
Phase 1: MVP基础框架 (7天)
├── 任务1.1: 基础模型和目录结构 (1.5天)
├── 任务1.2: 核心SkillManager实现 (2天)
├── 任务1.3: 增强安全扫描 (1.5天)
├── 任务1.4: 命令管理器MVP (1天)
└── 任务1.5: 集成测试和验证 (1天)

Phase 2: Agent集成和API扩展 (5天)
├── 任务2.1: LangGraph中间件集成 (2天)
├── 任务2.2: API和CLI扩展 (1.5天)
└── 任务2.3: 系统提示词和文档 (1.5天)

Phase 3: 高级功能 (6天)
├── 任务3.1: 技能版本管理 (2天)
├── 任务3.2: 性能监控和统计 (2天)
└── 任务3.3: 动态沙箱执行 (2天)

Phase 4: 测试和部署 (4天)
├── 任务4.1: 全面测试 (2天)
└── 任务4.2: 生产部署准备 (2天)
```

## 2. 详细任务分解（修正版）

### 任务1.1：基础模型和目录结构 (1.5天)

**目标**：建立简化的数据模型和目录结构，使用组合模式而非继承。

**子任务**：
1. **创建用户技能元数据模型** (0.5天)
   - `UserSkillMetadata`：磁盘元数据（文件路径、创建信息、质量评分、安全扫描结果）
   - `QualityScore`：质量评分模型（完整性、清晰度、结构、安全性、可复用性）
   - `ScanResult`：安全扫描结果模型

2. **创建组合式用户技能类** (0.5天)
   - `UserSkill`：组合现有`SkillDefinition`和`UserSkillMetadata`
   - 实现属性代理和转换方法
   - 保持与现有API的兼容性

3. **实现简化目录结构** (0.5天)
   - 扁平化技能存储目录：`~/.aiops/skills/{category}-{name-slug}/`
   - 技能ID生成规则：`{category}-{name-slug}`
   - 创建目录工具函数和验证逻辑

**输出文件**:
- `aiops/skills/user_models.py` (新建：用户技能相关模型)
- `aiops/skills/storage.py` (新建：目录结构管理)
- `tests/test_user_models.py` (新建：模型测试)

**依赖**: 无（独立模块）

**成功标准**:
- 用户技能模型可正确序列化/反序列化
- 目录结构符合扁平化设计
- 与现有`SkillDefinition`兼容

### 任务1.2：核心SkillManager实现 (2天)

**目标**：实现技能创建核心功能，包含质量门禁和原子写入。

**子任务**：
1. **实现基础SkillManager类** (1天)
   - `create_skill()`：完整创建流程（验证→质量评估→安全扫描→写入）
   - `_validate_new_skill()`：技能验证（名称、ID、目录存在性）
   - `_build_skill_content()`：构建完整SKILL.md（含YAML frontmatter）
   - `_atomic_write()`：原子写入机制（临时文件→重命名）

2. **集成质量评估门禁** (0.5天)
   - 集成`SkillQualityEvaluator`到创建流程
   - 设置质量门槛（默认0.7）
   - 提供详细的改进建议

3. **错误处理和异常体系** (0.5天)
   - `SkillCreationError`：通用创建错误
   - `SkillQualityError`：质量不达标错误
   - `SecurityBlockedError`：安全拦截错误
   - `SkillExistsError`：技能已存在错误

**输出文件**:
- `aiops/skills/manager.py` (新建：核心管理器)
- `aiops/skills/exceptions.py` (新建：异常类)
- `tests/test_skill_manager.py` (新建：管理器测试)

**依赖**: 任务1.1（基础模型）

**成功标准**:
- 技能创建全流程可通过
- 质量评估正确集成
- 原子写入确保数据完整性
- 错误处理覆盖所有场景

### 任务1.3：增强安全扫描 (1.5天)

**目标**：实现AIOps增强的安全扫描，包含专用规则和白名单。

**子任务**：
1. **移植基础威胁模式** (0.5天)
   - 从Hermes Agent移植基础规则（数据外泄、破坏性命令、持久化后门）
   - 实现正则模式匹配引擎
   - 创建基础`SkillsGuard`类框架

2. **添加AIOps特定规则** (0.5天)
   - 监控数据保护：`/var/lib/prometheus`等路径写保护
   - 服务操作检查：`systemctl stop`等命令识别
   - 配置防篡改：直接配置文件修改检测
   - 资源耗尽防护：`stress`、`dd`等工具检测

3. **实现白名单和上下文分析** (0.5天)
   - 白名单机制：允许特定模式（如`systemctl status`）
   - 上下文提取：显示匹配内容的上下文
   - 风险等级计算：dangerous/caution/safe三级

**输出文件**:
- `aiops/skills/guard.py` (新建：安全扫描器)
- `aiops/skills/security_patterns.py` (新建：威胁模式定义)
- `tests/test_skills_guard.py` (新建：安全测试)

**依赖**: 任务1.2（SkillManager）

**成功标准**:
- 基础威胁模式检测准确率>95%
- AIOps特定规则覆盖关键风险场景
- 白名单机制减少误报
- 扫描性能可接受（<500ms/文件）

### 任务1.4：命令管理器MVP (1天)

**目标**：实现基本的命令扫描和注册，支持渐进式披露。

**子任务**：

2. **实现渐进式内容加载** (0.5天)

   - 内容长度限制和截断
   - 按需加载完整内容机制

**输出文件**:
- `aiops/skills/commands.py` (新建：命令管理器)
- `tests/test_skill_commands.py` (新建：命令测试)

**依赖**: 任务1.1（目录结构），任务1.2（SkillManager）

**成功标准**:
- 命令扫描正确识别所有用户技能
- 渐进式内容加载减少token使用
- 命令响应时间<100ms
