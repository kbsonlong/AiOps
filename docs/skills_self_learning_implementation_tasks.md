# AIOps Skills 自学习系统实施任务分解

## 概述

本文档将《AIOps Skills 自学习系统设计》方案拆解为具体的可执行任务，包括优先级、工作量估计和依赖关系。

## 1. 总体实施路线图

```
Phase 1 (1-2周): 基础框架搭建
├── 任务1: 扩展技能数据模型 (2天)
├── 任务2: 实现 SkillManager 核心 (3天)
├── 任务3: 实现 SkillsGuard 安全扫描 (2天)
└── 任务4: 实现 SkillCommands 动态注册 (2天)

Phase 2 (1周): Agent 集成
├── 任务5: 扩展 RouterState 和节点 (3天)
├── 任务6: 更新系统提示词 (1天)
└── 任务7: 基础功能测试 (2天)

Phase 3 (1周): 高级功能
├── 任务8: 技能版本管理 (2天)
├── 任务9: 技能性能监控 (2天)
└── 任务10: API/CLI 扩展 (2天)

Phase 4 (3-4天): 测试与部署
├── 任务11: 端到端测试 (2天)
├── 任务12: 文档编写 (1天)
└── 任务13: 生产环境部署 (1天)
```

## 2. 详细任务分解

### 任务1: 扩展技能数据模型 (2天)

**目标**: 扩展现有的 `SkillDefinition` 模型，支持用户技能和自学习特性。

**子任务**:
1. 创建 `EnhancedSkillDefinition` 类 (4小时)
   - 继承现有 `SkillDefinition`
   - 添加用户技能特定字段：`skill_type`, `skill_file_path`, `skill_dir`, `created_by`, `created_at`, `updated_at`
   - 添加版本历史字段：`version_history`
   - 添加性能指标字段：`execution_count`, `success_rate`, `avg_execution_time`
   - 添加安全字段：`last_scan_result`, `risk_level`

2. 创建 `UserSkillMetadata` 模型 (2小时)
   - YAML frontmatter 对应的数据模型
   - 字段：`name`, `description`, `category`, `version`, `author`, `created_at`, `risk_level`, `platforms`, `dependencies`, `tags`

3. 创建安全扫描模型 (2小时)
   - `ScanResult`: 扫描结果模型
   - `ThreatPattern`: 威胁模式定义
   - `SecurityRiskLevel`: 安全风险等级枚举

4. 更新 `__init__.py` 导出 (1小时)

5. 编写单元测试 (3小时)
   - 测试模型序列化/反序列化
   - 测试字段验证逻辑
   - 测试继承关系

**输出文件**:
- `aiops/skills/models.py` (扩展)
- `aiops/skills/security_models.py` (新建)
- `tests/test_skill_models.py` (新建)

**依赖**: 无

### 任务2: 实现 SkillManager 核心 (3天)

**目标**: 移植 Hermes Agent 的 `skill_manager_tool.py` 核心逻辑，适配 AIOps 架构。

**子任务**:
1. 创建 `SkillManager` 类 (6小时)
   - 初始化：设置技能存储目录
   - `create_skill()`: 创建新技能（含原子写入）
   - `patch_skill()`: 局部更新技能
   - `delete_skill()`: 删除技能
   - `list_user_skills()`: 列出用户技能

2. 实现原子写入机制 (4小时)
   - `_atomic_write()`: 临时文件 -> 重命名策略
   - `_validate_skill_name()`: 验证技能名称格式
   - `_build_skill_directory()`: 构建技能目录结构

3. 实现技能文件解析 (4小时)
   - `_parse_skill_file()`: 解析 SKILL.md 文件
   - `_extract_frontmatter()`: 提取 YAML frontmatter
   - `_build_skill_definition()`: 从文件构建 EnhancedSkillDefinition

4. 实现错误处理 (3小时)
   - `SkillCreationError`: 技能创建异常
   - `SkillValidationError`: 技能验证异常
   - `SecurityBlockedError`: 安全拦截异常

5. 编写单元测试 (4小时)
   - 测试技能创建全流程
   - 测试原子写入正确性
   - 测试错误处理

**输出文件**:
- `aiops/skills/manager.py` (新建)
- `aiops/skills/exceptions.py` (新建)
- `tests/test_skill_manager.py` (新建)

**依赖**: 任务1（数据模型）

### 任务3: 实现 SkillsGuard 安全扫描 (2天)

**目标**: 实现技能安全扫描器，拦截危险代码。

**子任务**:
1. 创建 `SkillsGuard` 类 (4小时)
   - 初始化：加载威胁模式规则
   - `scan_skill()`: 扫描技能目录
   - `scan_content()`: 扫描文本内容

2. 定义威胁模式规则 (3小时)
   - 数据外泄模式：`curl.*\$ENV`, `wget.*\$ENV`
   - 破坏性命令：`rm\s+-rf\s+/`, `mkfs\.`
   - 持久化后门：`echo.*>>\s*~/(\.bashrc|\.zshrc)`
   - AIOps 特定风险：`service\s+stop`, `kill\s+-9`

3. 实现扫描结果报告 (3小时)
   - 风险等级计算：dangerous/caution/safe
   - 详细报告生成
   - 格式化输出

4. 集成到 SkillManager (2小时)
   - 在 `create_skill()` 后自动调用扫描
   - 危险技能自动删除
   - 记录扫描日志

5. 编写单元测试 (3小时)
   - 测试威胁模式检测
   - 测试扫描结果报告
   - 测试集成流程

**输出文件**:
- `aiops/skills/guard.py` (新建)
- `aiops/skills/security_patterns.py` (新建，威胁模式定义)
- `tests/test_skills_guard.py` (新建)

**依赖**: 任务2（SkillManager）

### 任务4: 实现 SkillCommands 动态注册 (2天)

**目标**: 实现动态技能命令扫描和注册，支持渐进式披露。

**子任务**:
1. 创建 `SkillCommandsManager` 类 (4小时)

   - `get_skill_content()`: 获取技能完整内容（渐进式披露）
   - `get_command_info()`: 获取命令信息

2. 实现命令解析 (3小时)
   - 解析 YAML frontmatter
   - 构建命令 slug：`name.replace(' ', '-').lower()`
   - 命令缓存机制

3. 实现渐进式披露 (3小时)
   - 仅加载主技能文件
   - 按需加载支持文件
   - 内容长度限制和截断

4. 集成到现有 CLI/API (3小时)
   - 扩展 `skill_cli.py` 支持用户技能
   - 扩展 `skill_api.py` 新增端点
   - 更新帮助文档

5. 编写单元测试 (2小时)
   - 测试命令扫描和注册
   - 测试渐进式内容加载
   - 测试 CLI/API 集成

**输出文件**:
- `aiops/skills/commands.py` (新建)

## 4. 风险与缓解措施

### 4.1 技术风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 安全扫描误报率高 | 中 | 高 | 逐步优化规则，提供误报反馈机制 |
| 技能执行性能下降 | 中 | 中 | 实现懒加载和缓存，监控性能指标 |
| 与现有功能冲突 | 低 | 高 | 充分测试兼容性，提供迁移工具 |
| 数据丢失风险 | 低 | 高 | 实现原子操作，定期备份，版本控制 |

### 4.2 实施风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|

## 5. 成功度量标准

### 5.1 技术指标
1. **技能创建成功率**: >95%
2. **安全扫描准确率**: 误报率 <5%，漏报率 <1%
3. **技能加载性能**: 平均加载时间 <500ms
4. **系统稳定性**: 无关键故障，可用性 >99.9%

### 5.2 业务指标
1. **用户采纳率**: >30% 的活跃用户使用技能功能
2. **技能复用率**: 平均每个用户技能被使用 >3 次
3. **问题解决效率**: 使用技能后平均解决时间减少 >20%
4. **用户满意度**: 用户评分 >4/5 分

### 5.3 质量指标
1. **技能平均质量评分**: >3.5/5 分
2. **技能更新频率**: 每月技能更新率 >10%
3. **社区贡献**: 外部贡献的技能数量每月增长
4. **知识积累**: 技能库覆盖 >80% 的常见运维场景
